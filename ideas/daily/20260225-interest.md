## 1. Claude Code Skills実践の日本での浸透 — 「設計・運用・自動化」が揃ってきた

はてブで同日にClaude Code Skills関連の3記事が高ブクマ（101/59/41 users）を記録。個別の「使い方」から**設計思想・自動化・スマホ単独開発**まで、実践知が一気に揃ってきた。02/16のinterestメモで取り上げた「Skills標準化」が、日本エンジニアコミュニティでも急速に根付いている。

### スキル設計の核心：Progressive Disclosure（101 users）
- **3段階の情報開示**: フロントマター（常時）→ SKILL.md本文（起動時）→ 補助ファイル（必要時のみ）
- **コンテキスト節約が目的**: 使わないスキルの情報はコンテキストに載せない = 02/16のAGENTS.md「20行制約」と同じ哲学
- **スクリプト配置の2戦略**:
  - スキル内配置 → 自己完結・再利用・配布向き
  - プロジェクトルート配置 → パイプライン構造向き（このプロジェクトが採用）
- **最重要な洞察**: ローカル実行だからこそ可能な機能（SharePoint接続、Playwright操作）がある。クラウドAIサービスでは実現できない領域がSkillsの真価

### コードレビューのナレッジ化：繰り返しをゼロに（41 users）
- **問題**: 「同じ指摘→修正→同じ指摘」の無限ループが開発現場で続いている
- **解決策**: 指摘内容をCLAUDE.md/Skillsに記録 → 「使い捨てコメント」から脱却
- **自動化パイプライン**: ローカルClaude Codeで自動修正 + GitHub Actionsでマージ時に自動実行
- **パラダイムシフト**: コードレビューは人間がAIに指摘するのでなく、**人間の知識をAIに学習させる場**へ

### スマホ単独開発の可能性（59 users）
- 著者「Cartera」ポートフォリオ管理アプリを**約2週間・ほぼスマホのみでApp Storeリリース**
- **成立する理由**: Claude Code on the Webはクラウド環境で動作するため「指示→放置」が可能。PR確認とマージのみ担当
- **課題**: Flutter→React Native移行（200コミット超の書き直し）、Pro($20)→Max($100)へのアップグレードが必要
- **App Store審査は「9分で一発承認」** — 品質が担保されていた証拠

### 02/16との接続
- 02/16: 「Skills標準化（Claude/Codex/Gemini）・ACP・klaw.sh」でエコシステム層が成熟
- 02/25: それを受けて**日本の実践者たちが「設計・運用・自動化」のノウハウを体系化**

## 3. Claude Codeの未来論争 — 「不要になる」vs「進化する」の大議論

r/ClaudeCodeで「Claude Codeはいずれ不要になる」（462 ups, **361 comments** — コメント数がupsより少ない激論の証拠）が投稿されたのと同日に、リモートコントロール機能追加、トークン65%削減ツール、tokfフィルタなど**Claude Code強化の話題が並立**。

### 「不要になる」論の中心的な主張
- **方向性**: IDE（Cursor等）がより深くAIを統合 → 専用エディタ不要になる
- **反論（コミュニティ多数派）**: Claude Codeは「エージェント実行環境」として機能しており、IDEの補完とは役割が異なる
- **361コメントの意味**: 開発者コミュニティがAIコーディングツールの未来を真剣に議論している温度感
- 02/16のinterestメモ「2026年もAIは開発者を置き換えられない理由」（183 ups/259 comments）の続編的議論

### 新機能：リモートコントロール対応（155 ups, 97 comments）
- **機能**: 別のマシンからClaude Codeを操作可能
- **意味**: スマホ単独開発（トピック#1）の可能性をさらに広げる。通勤中でもサーバー上のClaude Codeを制御できる
- **klaw.shやACPとの接続（02/16）**: エージェントをリモートで管理するインフラが整いつつある

### コスト最適化の実践
- **ローカル依存グラフによるトークン65%削減**（73 ups, 65 comments）: セッション間記憶保持も実現
- **tokfフィルタ**（94 ups, 41 comments）: Bash出力のトークン無駄を90%削減するCLIフィルタ
- **共通のテーマ**: Claude Codeの高コスト（Max $100/月）を前提に、いかにコンテキストを節約するかが実践課題

### IBM株急落との接続
- Claude CodeのCOBOL近代化能力が評価されIBM株が約25年ぶりの大幅下落（22 users）
- **これが「不要にならない」最大の根拠**: Claude Codeは既存システムへの侵食力を持ち、SaaS的なポジションを確立しつつある
- **vibe-codedのカオスで解雇（r/cscareerquestions, 636 ups）**: 無管理なAI活用の末路として、Claude Codeの適切な運用論にも繋がる

## 4. AWSのエージェント化加速 — Kiro/MCP/IAM Autopilotが収束

AWS側でも「エージェント向けAWS開発」のピースが揃ってきた。02/16のinterestメモで取り上げた「KiroのACP対応」から約1週間で、具体的な機能が次々と発表されている。

### IAM Policy Autopilot × Kiro（ワンクリックでIAMポリシー自動生成）
- **課題解決**: IAMポリシーは最小権限が正しいが、手動で書くのは煩雑でエラーが多い
- **解決策**: オープンソースの静的コード解析 → アプリが実際に使うAWSリソースを分析 → 最小権限ポリシーを自動生成
- **Kiro Power**: IDE内からワンクリックインストール（MCP設定不要）
- **エージェント開発との相性**: IAMポリシーを気にせず開発 → Autopilotが後から最適化

### AWS非機能要件ヒアリングシートの生成AI対応（30 users）
- **規模**: 5年ぶり更新、103問 → 206問に倍増（Claude Codeを活用して作成）
- **生成AI専用7問の新設**:
  - AIガバナンスポリシー、入出力データ保護
  - AI/MLワークロード固有のセキュリティ対策
  - 推論パフォーマンス指標（**TTFT/TPS/End-to-End Latency**）— これが特に重要
  - コスト管理方針、品質評価・テスト方針
- **TFTTとTPSが標準指標に**: 一般的なAWSシステム設計でも推論性能を定量評価する時代へ
- 著者の将来像：「プロンプトベースのスキルで対話的にヒアリング実施」= まさにClaude Code Skills活用
