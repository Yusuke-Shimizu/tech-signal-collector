# 興味関心メモ作成

トレンドネタ（`ideas/daily/YYYYMMDD-trend.md`）とユーザーの指示（要約済み記事など）を元に、興味関心メモ `ideas/daily/YYYYMMDD-interest.md` を作成する。必要に応じて `.claude/rules/user-profile.md` も更新する。

## 実行手順

### 0. 前提情報の読み込み

以下を読み込む:
- `.claude/rules/user-profile.md` — ユーザーの興味領域
- `ideas/daily/YYYYMMDD-trend.md` — 当日のトレンドネタ（存在する場合）
- 直前の興味関心メモ（前日分など）— 文脈の接続のため

### 1. 記事のピックアップと深掘り

ユーザーの引数: `$ARGUMENTS`

**引数がある場合 → 即座に深掘りへ進む**

引数の解釈ルール:
- **番号指定**（例: `1,3,7`）→ ★★★一覧の該当番号の記事を深掘り
- **`all`** → ★★★記事をすべて深掘り
- **URL指定** → そのURLの記事を深掘り
- **番号とURLの混在**（例: `1,3 https://example.com/article`）→ 両方を深掘り

**引数がない場合 → ★★★記事の一覧を提示して選択を求める**

トレンドメモから**興味度★★★の記事**を全ソース横断で抽出し、番号付きリストで提示する。

表示例:
```
今日の★★★記事（XX件）:

【はてブIT】
1. [タイトル](URL) (XXX users) - 一言メモ
2. ...

【Hacker News】
3. [タイトル](URL) (XXXpt) - 一言メモ
...

【Reddit】
7. [タイトル](URL) (XXX ups) - 一言メモ
...

深掘りしたい記事の番号を選んでください（複数可、例: 1,3,7）
追加したい記事のURLがあれば一緒にどうぞ
```

**深掘り処理**

選択された記事を **WebFetch で本文を取得して要約** する。
- 各記事の要点、主張、具体的なデータを抽出
- ユーザーの興味領域との関連ポイントを整理

### 2. 興味関心メモの作成

以下の方針で `ideas/daily/YYYYMMDD-interest.md` を作成:

**構成ルール**:
- トピックごとにセクション（`## 1. トピック名`）で整理
- 1メモあたり3〜6トピック程度
- 各トピックは以下を含む:
  - 複数ソース（はてブ、HN、Reddit、AWS等）の**横断的な分析**
  - ユーザーの興味領域との接続
  - 前日のinterestメモとの文脈接続（該当する場合）
  - 具体的な数字（ブクマ数、投票数、コメント数等）
  - 実践的なポイント・次のアクション

**分析の観点**:
- 同じテーマが複数ソースで同時に話題 → トレンドの確信度が高い
- コミュニティの温度感（コメント数が多い＝議論が活発）
- ユーザープロファイルの興味領域との関連度
- 「Deep Work思想」「エージェント設計」「ローカルLLM」など継続テーマとの接続

**文体**:
- 簡潔で情報密度が高い
- 箇条書き中心
- 太字（`**`）で重要ポイントを強調
- セクション間は `---` で区切り

### 3. ユーザープロファイルの更新

トレンドから新しい興味領域や関心の変化が見られた場合、`.claude/rules/user-profile.md` を更新する:
- 新しいキーワード・技術の追記
- 既存の記述の具体化・更新
- 不要になった項目の削除

**更新は保守的に**: 一時的な話題ではなく、継続的な関心が確認できた場合のみ。

### 4. 出力

**まず「興味関心メモ作成完了。」というメッセージを返してから、結果を保存。**

ファイル: `ideas/daily/YYYYMMDD-interest.md`

## 出力フォーマット

```markdown
# 興味関心メモ: YYYY-MM-DD

## 1. トピック名 — サブタイトル

導入文（なぜこのトピックが注目に値するか、複数ソースの横断的な状況）

### [サブトピック1のタイトル](元記事URL)（XX users / XX ups / XX pt）
- ポイント1
- ポイント2
- **重要な洞察**

### [サブトピック2のタイトル](元記事URL)
- ...

### 前日のinterestメモとの接続（該当する場合）
- ...

---

## 2. 次のトピック
...
```

## 注意事項

- **各サブトピックの見出しには必ず元記事のURLリンクを付ける**（例: `### [記事タイトル](https://example.com)`）。リンクなしの見出しは不可
- YYYYMMDDは実行日の日付を使用
- トレンドメモが存在しない場合は、先に `/neta-trend-daily` の実行を提案する
- ユーザーが特定の記事を指定した場合は、WebFetchで本文を取得して要約を含める
- プロファイル更新した場合は、変更内容をユーザーに報告する
- 前日のinterestメモが存在する場合、関連するトピックは文脈を接続する
