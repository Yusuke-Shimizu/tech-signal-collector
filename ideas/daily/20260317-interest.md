## 2. agency-agents — 144種類のエージェント定義で「役割分離」を即実装

### [Claude Codeに144種類のエージェントチームを作成 agency-agents活用](https://qiita.com/nogataka/items/5b5747f619e6eb745436)（32 users）

- GitHub Stars **40,889 / Fork 6,172**（2026年3月時点）
- **12カテゴリ×144エージェント**をMarkdownで定義。Claude Code / Copilot / Cursor / Gemini CLIに対応
- `./scripts/install.sh --tool claude-code --no-interactive` で `~/.claude/agents/` に一括配置
- **推奨ワークフロー**: Trend Researcher → Backend Architect → Frontend Developer → Security Engineer → **Reality Checker**
  - Reality Checker が品質ゲートとして機能。マイルストーンごとに「本当に要件を満たしているか」を検証
- 汎用エージェントとの差: 「React.memoでメモ化して」→ 単純提案 vs. プロファイリング→最適化→Core Web Vitals検証まで構造化

**自分の実践への示唆**:
- 現在の「Claude Codeとのコラボフレームワーク」（PLAN/SPEC/TODO/KNOWLEDGE.md）に役割特化エージェントを組み合わせると、**実装者エージェントと検証者エージェントを分離**できる

## 3. AWSエージェントインフラ — 長期記憶とサーバーレス開発が両輪で進化

AgentCore MemoryのGA進化と SAM×Kiro統合が同週に登場。エージェント実行基盤が「記憶」と「デプロイ」の2軸で同時に強化されている。

### [Amazon Bedrock AgentCore Memory ストリーミング通知GA](https://aws.amazon.com/about-aws/whats-new/2026/03/agentcore-memory-streaming-ltm/)（2026-03-12）

- メモリレコードの作成・更新を **Kinesisにプッシュ通知**（ポーリング不要）
- ダウンストリームワークフローのトリガー / アプリ状態更新 / 自動監査が即時可能
- **設計上のインパクト**: 長期記憶の変更をイベントとして扱える → エージェントの状態変化を他サービスと疎結合で連携できる
- 15リージョン対応（東京含む可能性高い）

### [SAM Kiro powerでサーバーレスアプリ開発を加速](https://aws.amazon.com/about-aws/whats-new/2026/03/aws-sam-kiro-power/)（2026-03-13）

- Kiro IDE上でSAMのプロジェクト初期化・ビルド・デプロイ・ローカルテストをAIエージェントが支援
- EventBridge / MSK / Kinesis / DynamoDB Streams / SQS対応 → **イベント駆動パターンをAIが設計段階から提案**
- 動的にガイダンスをロード（RAGに近い設計）→ Kiro自体がエージェント設計のリファレンス実装になっている

**AgentCore Memory × SAM Kiro Powerの組み合わせ**:
- エージェントが長期記憶を更新 → Kinesisで通知 → SAMのイベント駆動Lambdaが反応
- このパターンが「エージェントがインフラを動かす」という次世代アーキテクチャの基盤になりうる
