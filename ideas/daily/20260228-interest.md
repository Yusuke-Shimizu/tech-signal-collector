# 興味関心メモ: 2026-02-28

## 1. マルチエージェント実践 — オーケストレーションとボトルネック排除の設計論

今日はClaude Codeのエージェント設計に関する質の高い実践記事が複数ソースで同時に出た。ユーザープロファイルの最重要領域に直撃する日。

### #1 Claude Code + Codexオーケストレーション（121 users）
- **実装→審査→修正のループをYAMLで宣言的に制御**する"TAKT"というOSSを開発・公開
- `edit: false` でレビュアーエージェントの改変を物理的に禁止し、審査の純粋性を担保
- "Lost in the Middle"対策としてタスク単位でコンテキストを分離
- 30分〜2時間の自走で洗い物ができるレベルに到達
- **実践可能性が高い**: npmで即試せるOSS。自分のプロジェクトに導入候補

### #2 Agentワークフローのボトルネック排除 Skill設計（71 users）
- **Skillのトリガー文に「聞きたくなった瞬間」を書く**という逆転の発想
  - "Before asking the user any question...when unsure how to proceed" → 自動ロードで自己解決へ誘導
- 自己検証のツールセット: chrome MCP、screenshot、read_console_messages、read_network_requests
- 人間を呼ぶのは物理操作のみ（SMS認証、CAPTCHA）。呼ぶときは選択肢2〜4個に限定
- **Ralph Loopとの統合**: 外側ループ（コンテキスト制約対応）＋内側ループ（自律判断）

### #5 エージェント時代の組織・開発論（151 users）
- GitHubのコミットの4%がすでにClaude Code起源。年末には20%超の予測
- 個人の生産性天井: daily 90 commit / 30,000行 → 理論上 300 commit / 60,000行
- ボトルネックは人間側の「指示待ち・検収待ち・対応荒れ」の3つ
- **推奨ドキュメント戦略**: CLAUDE.md × 1 + ADR群。コード説明ドキュメントは変更速度に追従不可で不要
- 推奨チーム構成: アイデアマン(1) + ランナー(1) + レビュアー群(3+)

### 3記事の横断分析
| テーマ | #1 TAKT | #2 Skill設計 | #5 組織論 |
|--------|---------|-------------|----------|
| 人間ボトルネック | エージェントに代替 | Skillで自律化 | 組織設計で解消 |
| コンテキスト管理 | タスク単位で分離 | 外側/内側ループ | CLAUDE.md + ADR |
| 自己検証 | review→fixループ | MCPで自動検証 | 完全自動化は危険 |

**共通の示唆**: 「AIに聞かせない設計」がAgentワークフローの核心

## 3. AIと「始めない」能力 — Deep Work思想との接続

### #20 「AIはプロジェクトを始めさせすぎる」（59 users）
- AIによって新規プロジェクト立ち上げコストが激減→完成しないサイドプロジェクトが際限なく増殖
- 困難なメインプロジェクトを後回しにして「楽しい新規プロジェクト」に逃げる傾向
- 従来は立ち上げコストが自然なフィルターとして機能していた

**Deep Work思想との直結**
- 「AIが定型作業を処理する分、人間は深く考える時間を確保すべき」という思想の裏返し
- AIが「始めること」を容易にしすぎると、深く考える対象が拡散する
- **「始めない」能力、「終わらせる」希少資源の管理がこれからのメタスキル**
- 今日のGitHub Actions修正作業も: 「とりあえず動かしてみる」コストが下がった分、試行錯誤の頻度が上がった
